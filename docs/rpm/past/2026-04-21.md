# 2026-04-21 — rpm

## Backlog ordering + session-end evolution (v2.7.7 → v2.7.9)

Three consecutive patch releases in one session, tightening how the
rpm backlog represents current state and how `/session-end` ceremoniously
handles clean sessions.

## Accomplished

- **v2.7.7 — execution-order reframe.** Replaced the "priority-ordered"
  framing of `tasks.org` with **execution order** (top = next to do,
  not an abstract priority score; "chronologically" = when work needs
  to happen, not when it was created). Added a 4-band sort within
  each `* Parent` group: Actionable → Blocked → Postponed → Historical.
  Blocked/postponed items auto-drift to the bottom of their band
  whenever the file is written. Touched `plugin/skills/backlog/SKILL.md`,
  `plugin/skills/session-end/SKILL.md` Phase 3b (auto-demote sweep
  + mismatch check), memory `feedback_tasks_execution_order.md`,
  MEMORY.md index. One-time re-sort of `tasks.org` (15/15 task IDs
  preserved, verified by ID-set diff).

- **v2.7.8 — pivot-capture rule.** On mid-session user pivot to
  TaskCreate-worthy work (multi-step, multi-file, or explicit
  "work on X"), LLM immediately (1) edits `docs/rpm/~rpm-session-start`
  to update the `task:` field, (2) inserts `** TODO` at the TOP of
  the actionable band in `tasks.org`. No user question — the ask IS
  the confirmation. Skipped for tactical single-step follow-ups.
  Codified in `plugin/hooks/_directives.sh` (shared SessionStart +
  post-compact injection), `plugin/skills/backlog/SKILL.md` preamble,
  memory `feedback_pivot_capture.md`. Dogfooded twice this session
  (trim-session-end pivot, pivot-capture-rule pivot itself).

- **v2.7.9 — session-end trim.** Killed the Phase 1 "Actions
  (pick any, multiple OK)" outer menu — redundant with Phase 2
  sub-prompts, which already fire only when their surface has
  content. Added a **fast-path** single-response mode: when drift /
  learnings / mismatch all empty and git clean after the auto-apply
  commit, emit one compact `## Session end (rpm <version>)` block
  (Accomplished / Tracker updates / What's next / handoff) with
  Phase 4 cleanup inlined. Full 4-phase flow preserved as fallback
  whenever any decision surface is non-empty.

- **Backlog follow-ups filed.**
  - `publish-all-push-dev` — fold `git push dev master` into
    `publish-all.sh` as a pre-step so "release" means "both remotes".
  - `pivot-capture-hook` — optional PostToolUse enforcement that
    mirrors `~rpm-session-start` `task:` changes to `tasks.org` top
    automatically; defer until LLM-side drift is observed.

## Key Discoveries

- **`publish-all.sh` pushes only to `plugin`.** The clean-tree
  precondition means you have to `git push dev master` manually
  before every release. Captured as backlog item
  `publish-all-push-dev`.

- **Skill bodies are cached at SessionStart.** The `/session-end`
  invocation that ended this session is running the pre-trim body
  (with Actions menu) because v2.7.9 was committed mid-session.
  Next session will pick up the trim. Worth keeping in mind
  whenever we change skill behavior — the current session can't
  self-test it.

- **`scan.sh` still emits `version=unknown`** when invoked from
  skill-body bash context because `$CLAUDE_PLUGIN_ROOT` isn't set
  there. Known issue; backlog entry `scan-version-fallback`.

## What Didn't Work

- Parallel `Edit` calls to an unread file fail with `"File has not
  been read yet"`. `MEMORY.md` needed an explicit `Read` before
  `Edit`; I missed it in one batch and had to recover. Consider a
  session-start precache of frequently-edited files, or just an
  internal habit.
