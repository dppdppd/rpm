### Session-end UX improvements (`b50423a`)

User feedback drove three changes: (1) omit menu items with empty
preconditions, (2) questions only at end of responses as numbered
menus, (3) clear/restart instructions as the very last output after

### Session-start scan.sh auto-inject (2.1.3, `c17dbcf`)

Created `skills/session-start/scripts/scan.sh` — bundles latest
past-file lookup, git status, stash list, and PRESENT.md drift
check into a single script that auto-injects before the skill body
loads. Cuts session-start from ~5 sequential tool-call rounds to ~2.
Same pattern session-end already uses.

### Session-end latency optimizations (`f605ead`)

Three round-trip reductions: (1) pre-read today's past log in
Phase 1b so Phase 2 writes are truly parallel, (2) merge Phase 2
commit + Phase 3 presentation into one response, (3) merge Phase 5
rm + handoff into one response. ~4 rounds instead of ~6.

### /pm:audit quick — 1 finding fixed (`074326c`)

Ran audit quick mid-session. One finding: daily log missing for
2026-04-10 (2 commits landed, no log). Created log + appended
audit marker to PM-LOG.md.

## Key discoveries

- **Session-start latency was dominated by sequential tool-call
  rounds**, not LLM thinking. The `ls` to find the latest past
  file, `git status`, `git stash list`, and the two-step drift
  check were 5 round trips that could be collapsed to zero with
  the same auto-inject pattern session-end already uses.
- **Session-end had a hidden past-log dependency**: Phase 2 says
  "write all three in parallel" but if today's past log exists,
  it needs reading before appending. Making the read explicit in
  Phase 1b removes the hidden dependency.

# Session 3 — Competitive audit, hook-driven lifecycle, rename to rpm

## Accomplished

### Competitive audit against 8 plugins
Researched flow-next (566 stars), claude-mem (46.9k), pro-workflow
(1.8k), and 5 smaller plugins. Identified 4 tiers of features to
add based on competitor strengths. Unique to pm: structured doc
auditing, past/present/future lifecycle, deep research, zero-cost
scan.sh drift detection.

### Hook-driven session lifecycle (2.2.0, `c53f699`)
Replaced skill-driven session lifecycle with 5 hooks:
- **SessionStart** → `session-start-auto.sh`: auto-injects PM context
- **PreCompact** → `pre-compact.sh`: checkpoints to daily log
- **PostCompact** → `post-compact.sh`: re-injects recovery state
- **Stop** → `stop-learn-capture.sh`: captures learning signals to JSONL
- **UserPromptSubmit** → `prompt-session-nudge.sh`: wrap-up nudge at ~90min

Deleted 3 skills: `session-start`, `session-update`, and the
planned `context-scouts` (incompatible with auto-start). Added
structured task dependencies (:ID:/:BLOCKED_BY: in FUTURE.org)
with scan.sh validation and cycle/dangling-ref detection.

### Rename to rpm (`d443ba6`)
Plugin renamed from "pm" to "rpm" — Relentless Product Manager.
Double meaning: relentless PM + revolutions per minute. All command
prefixes (`/pm:` → `/rpm:`), skill directory (`skills/pm/` →
`skills/rpm/`), and agent namespace (`pm:auditor` → `rpm:auditor`)
updated. `docs/pm/` directory and `~pm-*` filenames kept (they
refer to the PM function, not the plugin namespace).

### Rebranded copy
README, SKILL.md, CLAUDE.md, PM.md all rewritten around the
product manager persona: "Tracks what shipped, what's next, and
what's drifting — so you can focus on building."

## Key discoveries
- PreCompact/PostCompact stdout goes to debug log, not context —
  need CLAUDE.md fallback instruction for post-compaction recovery
- Stop hook receives `last_assistant_message` in stdin payload
- Bash variable names can't contain hyphens — org-mode IDs need
  sanitization via `tr '-' '_'`
- "Product Manager" maps perfectly as a persona for what the tool does

## What didn't work
- Context scouts (opt-in codebase scanning at session-start) — cut
  because it contradicts the auto-start automation goal

## Next
- Dogfood the new hooks in a fresh session (auto-start, Stop
  capture, PreCompact checkpoint)
- Fix 3 broken refs from the rename (skills/pm/ → skills/rpm/)
