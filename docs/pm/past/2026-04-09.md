## Key discoveries

- **Commands-to-skills unification is official.** Anthropic's
  `code.claude.com/docs/en/skills` explicitly says *"Custom commands
  have been merged into skills."* Tracking issue
  `anthropics/claude-code#37447` is open but stale-labeled with no
  Anthropic response on a deprecation schedule. Commands still work
  indefinitely; skills are the recommended path for new work.
- **ccpm is the ecosystem signal.** They migrated FROM the exact
  same `/pm:*` slash command prefix TO an Agent Skill, preserving
  legacy on a `v1` branch. Strongest single data point justifying
  option C on the S1 decision. Their README: *"Looking for v1? —
  The original `/pm:*` Claude Code slash command system is
  preserved on the `v1` branch."*
- **`disable-model-invocation: true` removes the description from
  context.** Forced a redesign of Phase 4's session-end skill: if
  the flag is set, Claude doesn't know the skill exists and can't
  recommend it proactively. Solved by dropping the flag and adding
  a pre-flight auto-invocation gate in the skill body that asks
  for confirmation before running Phase 2 (auto-commits).
- **Plugin agent frontmatter schema is tighter than assumed.**
  Only `name`, `description`, `model`, `effort`, `maxTurns`,
  `tools`, `disallowedTools`, `skills`, `memory`, `background`,
  `isolation` are officially listed in `plugins-reference`. Common
  keys like `color` and `whenToUse` are silently ignored —
  cosmetic debt rather than a bug.
- **WebFetch against `github.com` URLs is denied in this
  environment.** User added a `CLAUDE.md` guardrail mid-session:
  use `curl --max-time 30 -sL` via Bash instead. `gh api` also
  works for GitHub URLs. Switched outward-research fetches to
  `gh api` and continued.

## What didn't work

- **First attempt at outward research via WebFetch failed.** The
  tool was denied for `github.com/anthropics/claude-code/issues/37447`
  — user cancelled it and told me to move on. Switched to `gh api`
  for subsequent GitHub fetches. The WebFetch guardrail landed in
  `CLAUDE.md` after the fact.
- **Initial A/B/C option listing was not blank-line-separated.**
  User flagged the format mismatch against the plugin's own Findings
  Menu rule ("blank line between options"). Saved new feedback
  memory (`feedback_option_spacing.md`) and re-printed with proper
  spacing.
- **A3 auto-invocation policy didn't survive its first review.**
  Initial A3 had `disable-model-invocation: true` on `session-end`.
  User followed up saying the model should proactively recommend
  session-end when context gets long. `disable-model-invocation`
  blocks that behavior at the description-loading level, so I
  redesigned to A3' (drop the flag, add a pre-flight gate in the
  skill body). Caught before the Phase 4 commit landed.
