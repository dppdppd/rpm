### Session 3 — stale-session detection + proactive marker (v2.5.2)
- First approach: detect stale marker via `SOURCE=startup` gate.
  Wrong gate — user's `c` wrapper uses `--continue` by default,
  which emits `SOURCE=resume`, so detection never fired despite
  the leftover marker being real.

  hook's stdin payload and the marker's stored session_id.

  new CC processes (`--continue`, cold start) get a new

### Session 5 — paired-marker redesign + rename + softer stale UX
- Bug repro: ran `/session-end` in prior CC process, then `/clear`
  in the same process — the proactive marker block recreated
  `~rpm-session-active` with `task:(unassigned)`, and the next CC
  process flagged it as "didn't wrap up". volta hit the same
  pattern. Two failed attempts first: (1) work-evidence heuristic
  (false-positive on uncommitted past/ log after session-end),
  (2) last-session.session_id piggyback (still had legacy-state
  bug).
- Final design: **paired markers named after their events**.
  `~rpm-session-active` → `~rpm-session-start` (SessionStart
  event). New `~rpm-session-end` created by `/session-end`
  skill. Pair on session_id → clean handoff → silent rm.
  Unpaired start → stale. Orphan end (no start) → cleaned.
  End marker matching current CC session preserved across
  `/clear` transition in same process.
- Renamed across all hooks, skill, tests, .gitignore.
- Softened stale warning: "rpm: your previous session wasn't
  ended with /session-end" + offer `/resume` or clear.
  Task menu no longer shown on stale path.
- 26 bats tests (5 new for pair logic), shellcheck clean.

## Key Discoveries
- The old date-compare stale heuristic (`LAST_ENDED < MARKER_STARTED`)
  can't distinguish "orphan marker from /clear-after-session-end"
  from "real stale from crash" — both match the predicate.
  Deterministic signal (paired session_id markers) is required.
- Work-evidence heuristics are leaky: `/session-end` modifies
  `past/YYYY-MM-DD.md` and if uncommitted it registers as "work"
  even though the session ended cleanly.

## What Didn't Work
- Heuristic: check commits/dirty-tree/jsonls since marker.started —
  false-positive when user didn't commit session-end's past/ updates.
- Piggyback: add `session_id:` to `~rpm-last-session` — doesn't fix
  legacy state and is less clean than a dedicated handoff file.
