## Tasks

### Task 1: Fix auditor.md session drift path
- **Severity:** High
- **Dimension:** Architecture
- **What's wrong:** `agents/auditor.md` line 57 instructs the auditor
  to find session JSONL files at
  `~/.claude/projects/$(pwd | sed 's|/|-|g; s|^-||')/*.jsonl`. The
  `s|^-||` strips the leading dash, but the actual directory name
  retains it (e.g., `-home-coder-projects-rpm`). The auditor looks
  in `home-coder-projects-rpm` (without dash) and finds nothing.
  Session drift checking silently fails on every audit.
- **Why it matters:** Session drift detection is a core auditor feature
  (Section 8 of the audit report). Without it, unjustified changes go
  unreviewed. This bug has been present since the auditor was created.
- **Research says:** Inward — the Claude Code project directory naming
  convention replaces `/` with `-`, preserving the leading dash from
  the absolute path prefix. Verified against actual filesystem.
- **Fix:** In `agents/auditor.md` line 57, change:
  `$(pwd | sed 's|/|-|g; s|^-||')` to `$(pwd | sed 's|/|-|g')`
- **Effort:** Small

### Task 4: Fix scan.sh broken_refs false positives
- **Severity:** Medium
- **Dimension:** LLM Workflow
- **What's wrong:** `skills/session-end/scripts/scan.sh` broken_refs
  section (line 104) filters shell metacharacters with
  `grep -qE '\$|\*|\||>|~'` but does not include `{` or `}`. Template
  patterns like `plugin/skills/{name}/SKILL.md` in context.md pass
  through and generate false-positive broken-ref findings on every
  scan. Additionally, paths relative to `docs/rpm/` (like
  `future/tasks.org` in context.md) fail project-root resolution.
- **Why it matters:** scan.sh runs at every session-end and every
  `/audit quick`. False positives are noise that trains users to
  ignore findings.
- **Research says:** Observed in the Phase 1a scan output. The
  `status.md` Known Issues section already documents this.
- **Fix:** Add `{` and `}` to the metacharacter filter on line 104:
  `grep -qE '\$|\*|\||>|~|\{|\}'`. For the relative-path issue,
  consider also resolving paths relative to the source file's
  directory, or exclude known-relative entries via a comment.
- **Effort:** Small
