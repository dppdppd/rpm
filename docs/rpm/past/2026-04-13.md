### Session 3 — stale-session detection + proactive marker (v2.5.2)
- First approach: detect stale marker via `SOURCE=startup` gate.
  Wrong gate — user's `c` wrapper uses `--continue` by default,
  which emits `SOURCE=resume`, so detection never fired despite
  the leftover marker being real.
- Redesigned: detect via **session_id mismatch** between the
  hook's stdin payload and the marker's stored session_id.
  `/clear` and `/resume` within the same CC process preserve
  session_id, so they correctly stay on the resume path;
  new CC processes (`--continue`, cold start) get a new
  session_id and trigger stale detection.
- Hook now writes a proactive `~rpm-session-active` marker on
  every fresh session (task: `(unassigned)`), guaranteeing any
  work done without picking from the menu is still visible to
  the next session.
- Task-selection instruction changed: Claude `Edit`s the
  marker's task field instead of rewriting the whole file —
  preserves session_id and started: for accurate stale
  detection later.
- Stale recommendation output is copy-friendly (no single
  quotes around `/resume <id>` / `/session-end`) and leads with
  "I STRONGLY recommend".
- `/session-end` Phase 1e: if marker task is still
  `(unassigned)` at wrap-up, auto-derive a title from the
  session's work (commits, modified files, conversation).
- 21 bats tests (6 new across the two sessions).
- Bumped plugin.json to v2.5.2; tagged and pushed.
