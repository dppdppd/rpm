## Key discoveries

- **Claude Code plugin schema puts the manifest in
  `.claude-plugin/plugin.json` exclusively.** A root-level mirror
  is not part of the spec — it was pure maintenance burden.
- **`agents/auditor.md` was dead code** — registered as
  `pm:auditor` but never invoked by its own command. The inline
  prompt in `/pm:audit` Standard Phase 1 had already diverged from
  the agent's spec (auditor had 8 short bullets; audit.md had 8
  detailed bullets).
- **Heavy audit's biggest failure mode is over-indexing on inward
  research.** Caught only because the user flagged it manually.
  Command wording now makes outward research required and distinct.
- **Claude Code ships native persistent Tasks** (`~/.claude/tasks/`)
  since Jan 2025 with deps, blockers, and `activeForm` spinner. We
  were duplicating a worse version in `FUTURE.org` alone. Hybrid
  integration done.
- **Competitive landscape snapshot** (from Phase 3 outward research)
  saved in `docs/pm/reviews/2026-04-08.md`: `claude-sessions`,
  Superpowers, Cline Memory Bank, Docs Guardian, native Tasks, Spec
  Kit, Kiro.

## What didn't work

- The first Phase 2 (original Heavy mode wording: "Research") was
  purely inward — validated against Anthropic docs but never
  looked at real alternatives. The user had to redirect the review
  toward competitive analysis. This drove Finding 8 (restructure)
  and is why outward research is now marked REQUIRED.
- One parallel `Edit` to `docs/pm/reviews/2026-04-08-plan.md`
  failed on the first attempt due to an `old_string` mismatch
  (text from a stale mental copy vs the actual file content).
  Retried successfully with the exact on-disk text. Lesson: when
  the same file is hit by two edits in one parallel batch, second
  edit may race against the state I assumed.
