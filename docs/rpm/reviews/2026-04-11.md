## Phase 3: Outward Research

### Competitors analyzed

| Tool | What it does | Better than rpm | rpm does better | Verdict |
|------|-------------|-----------------|-----------------|---------|
| **ccpm** (automazeio) | Spec-driven PM with PRD→Epic→Issue→Code chain, 14 bash scripts, agentskills.io standard | Full planning pipeline, harness-agnostic, parallel agents in worktrees, GitHub Issues as source of truth | Doc drift detection, hook automation, audit scoring, zero dependencies | Complementary |
| **claude-mpm** (bobmatnyc) | Multi-agent PM with 47 agents, 56 skills, MCP integrations, SDK runtime mode | Massive scope (Google Workspace, Slack, Notion), semantic code search, context-pressure nudging, monitoring dashboard | Zero dependencies, focused scope, actual audit system with scored findings, simplicity | Complementary |
| **product-forge** (jpoutrin) | PRD management + specialized technical agents (Django, FastAPI, React, etc.) | Documentation-first approach (agents research docs before coding), MCP security framework, specialized agents | Session lifecycle, doc drift detection, audit system, lightweight | Complementary |
| **claude-sessions** (maleta) | Session tracking via SESSION_SUMMARIES.md + Haiku auto-summarization + web UI | Very simple, auto-summarization, web browsing history | Everything else — task tracking, audit system, doc drift, hook automation | Complementary |

**Key takeaways:**

1. **Market has grown.** claude-mpm and product-forge are significant
   new entrants since the 2026-04-10 audit. The Claude Code plugin
   ecosystem is maturing rapidly.
2. **rpm's niche is intact.** No competitor combines doc drift detection,
   hook-driven session lifecycle, and scored audits. The closest is
   cc-spex (drift detection only) from the prior audit.
3. **Simplicity is a feature.** claude-mpm requires Python, pip install,
   60+ services, 100+ CLI commands. rpm is pure markdown + bash with
   zero dependencies. For solo developers, this is a competitive
   advantage.
4. **Context-pressure nudging.** claude-mpm nudges at 70%/85%/95%
   context thresholds. rpm nudges at 90 minutes. Context-pressure is
   more accurate but requires SDK-level integration that a pure
   markdown+bash plugin can't access. Time-based is a pragmatic
   alternative. No action needed.
5. **Harness-agnostic standard.** ccpm adopted agentskills.io. rpm
   is Claude Code-specific. Adopting the standard would expand reach
   but sacrifice hook-driven automation (the core differentiator).
   Deferred — clear tradeoff.
