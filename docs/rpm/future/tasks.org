*** DONE opencode port: distribution (subtree-split to plugin/opencode)
    CLOSED: [2026-04-18]
    :PROPERTIES:
    :ID: opencode-port-distribution
    :END:
    - Chose subtree-split over npm: the npm route ships only the
      plugin module, while rpm's commands/skills/agents must land
      in the user's project .opencode/ dir. A single `opencode`
      branch with everything pre-baked avoids a post-install dance.
    - `scripts/publish-opencode.sh` stages gitignored mirrors on a
      throwaway branch, subtree-splits, pushes to
      plugin/opencode (force). Supports --dry-run for verification
      and alternate remotes (dev/opencode for testing).
    - `opencode/install.sh` lives at the root of the split; users
      curl+bash it from
      https://raw.githubusercontent.com/dppdppd/rpm/opencode/install.sh
      to copy .opencode/ into their project.
    - Dry-run verified: 38 files in split, install.sh + README.md +
      package.json at root, full .opencode/ tree underneath.
    - Publish not yet run (awaiting maintainer go-ahead).
